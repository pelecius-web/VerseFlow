You are an independent auditor tasked with signing off on a production-hotfix. The original debug report, the fix plan, and the committed changes are all available. Your job is to perform a thorough, evidence-based audit covering three areas: A) Correctness — does the fix resolve the root cause exactly, and is it minimal? B) Safety — is there any indication of new bugs, regressions, or technical debt introduced? C) Testability — was the fix accompanied by appropriate regression tests (or at least clear manual verification steps)? For B, you must enumerate existing features that share data paths, user flows, or code modules with the changed area, and confirm they have been explicitly verified. Produce a final audit report with sections: Findings, Regression Risk Items, Test Coverage Assessment, and Sign-off Recommendation (Approved / Conditionally Approved / Rejected). If conditionally approved, list the precise conditions to meet before full approval. The report should serve as the last checkpoint before the fix reaches users.
